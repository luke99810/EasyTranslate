## 一、产品概述

### 1.1 产品定位

**一句话描述**：免费开源的学术PDF在线翻译工具，专注解决公式保留和排版保持问题。

**核心价值主张**：
- 网页即用，无需安装
- 本地处理，保护隐私
- 公式不乱，排版不散
- 双语对照，阅读高效

### 1.2 产品愿景

成为学术界最受欢迎的免费PDF翻译工具，让语言不再成为科研的障碍。

### 1.3 目标用户

#### 主要用户群体
1. **研究生（占比50%）**
   - 年龄：22-28岁
   - 需求：大量阅读英文文献，预算有限
   - 痛点：现有工具收费高或效果差

2. **科研人员（占比30%）**
   - 年龄：28-45岁
   - 需求：专业领域深度阅读
   - 痛点：论文隐私安全，专业术语翻译

3. **高校教师（占比20%）**
   - 年龄：30-55岁
   - 需求：备课、科研双重需求
   - 痛点：批量处理、双语对照

## 二、用户故事

### 2.1 核心用户故事

#### US-001：快速翻译论文
> 作为一名研究生，
> 我希望上传PDF后能快速得到翻译版本，
> 这样我可以高效阅读英文文献。

**验收标准**：
- 支持拖拽上传PDF文件
- 翻译时间不超过原文档页数×5秒
- 支持显示翻译进度

#### US-002：保留公式排版
> 作为一名理工科研究生，
> 我希望翻译后的文档能保留数学公式，
> 这样我可以理解论文的技术内容。

**验收标准**：
- LaTeX公式完整保留
- 公式编号不乱
- 公式可正常渲染显示

#### US-003：双语对照阅读
> 作为一名科研人员，
> 我希望能够中英对照阅读，
> 这样我可以核对翻译准确性。

**验收标准**：
- 支持段落级双语对照
- 支持点击原文跳转到译文
- 支持调整对照布局（左右/上下）

#### US-004：本地处理保护隐私
> 作为一名科研人员，
> 我希望文件在本地处理不上传云端，
> 这样我的研究内容不会泄露。

**验收标准**：
- 明确告知用户数据处理方式
- 不存储用户上传的文件
- 翻译完成后自动清理临时文件

#### US-005：导出翻译结果
> 作为一名教师，
> 我希望下载翻译后的PDF或Word，
> 这样我可以保存或分享给学生。

**验收标准**：
- 支持导出PDF格式
- 支持导出Word格式（保留可编辑性）
- 导出文件保留原文档结构

### 3.2 功能详细说明

#### 3.2.1 文件上传模块

**输入**：
- 文件格式：PDF
- 文件大小：最大50MB
- 页数限制：最大100页（MVP版本）

**处理流程**：
1. 用户上传PDF文件
2. 前端校验文件格式和大小
3. 上传至后端临时存储
4. 返回文件ID和处理状态

**错误处理**：
- 格式错误：提示"请上传PDF格式文件"
- 大小超限：提示"文件大小超过50MB限制"
- 损坏文件：提示"无法读取PDF文件，请检查文件是否损坏"

#### 3.2.2 翻译处理模块

**处理流程**：
1. 解析PDF结构（文本、公式、图片）
2. 提取可翻译文本块
3. 调用翻译API进行翻译
4. 重组文档结构
5. 生成翻译后的文档

**技术要求**：
- 公式识别：使用正则表达式识别LaTeX公式
- 图片处理：保持原始位置和大小
- 段落保持：保持原文段落结构

#### 3.2.3 双语对照阅读模块

**布局模式**：
- 左右对照：原文左，译文右
- 上下对照：原文上，译文下
- 单栏模式：仅显示译文

**交互功能**：
- 点击段落高亮对应译文
- 同步滚动
- 字体大小调节（12px-24px）

#### 3.2.4 导出模块

**PDF导出**：
- 保留原文档排版
- 嵌入中文字体
- 支持选择导出范围（全部/当前页）

**Word导出**：
- 保留可编辑性
- 保留图片和公式
- 支持.docx格式

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| 上传速度 | <5秒（10MB文件） | 正常网络环境下 |
| 翻译速度 | <5秒/页 | 单页翻译处理时间 |
| 页面加载 | <3秒 | 首屏加载时间 |
| 并发处理 | 支持5个并发 | MVP版本 |

### 4.2 兼容性需求

| 平台 | 支持版本 |
|-----|---------|
| Chrome | 最新2个版本 |
| Firefox | 最新2个版本 |
| Edge | 最新2个版本 |
| Safari | 最新2个版本 |

### 4.3 安全需求

1. **数据传输**：使用HTTPS加密
2. **文件处理**：临时文件定期清理
3. **API安全**：用户API Key前端加密存储
4. **隐私保护**：不存储用户上传的任何文件

### 4.4 可用性需求

1. **界面语言**：简体中文
2. **错误提示**：友好的中文错误提示
3. **操作反馈**：关键操作有加载状态和成功提示
4. **帮助文档**：提供使用说明和常见问题

## 六、技术约束

### 6.1 前端约束

- 使用React 18+
- 使用TailwindCSS进行样式开发
- 支持响应式布局
- 不使用jQuery

### 6.2 后端约束

- 使用Python 3.10+
- 使用FastAPI框架
- 异步处理翻译任务
- 支持Docker部署

### 6.3 依赖约束

- PyMuPDF：PDF解析
- pdf2image：PDF转图片（OCR场景）
- requests：HTTP请求
- 翻译API客户端库
